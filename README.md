# Управление учетными записями

Vue.js 3 приложение для управления учетными записями с использованием Composition API, TypeScript и Pinia.

## Функциональность

- Добавление новых учетных записей
- Редактирование существующих учетных записей
- Удаление учетных записей
- Валидация полей формы
- Сохранение состояния в Pinia (сохраняется при обновлении страницы)
- Адаптивный интерфейс

## Структура учетной записи

Каждая учетная запись содержит:

1. **Метка** (необязательное поле)
   - Максимум 50 символов
   - Текстовые метки разделяются через `;`
   - Сохраняется как массив объектов: `[{ text: "метка1" }, { text: "метка2" }]`

2. **Тип записи** (обязательное поле)
   - `LDAP` - скрывает поле пароля, сохраняет пароль как `null`
   - `Локальная` - показывает поле пароля

3. **Логин** (обязательное поле)
   - Максимум 100 символов

4. **Пароль** (обязательное поле только для типа "Локальная")
   - Максимум 100 символов
   - Скрывается для типа "LDAP"

## Технологии

- **Vue.js 3** с Composition API
- **TypeScript**
- **Pinia** для управления состоянием
- **Vuetify** как UI фреймворк
- **Vee-Validate** для валидации форм
- **Vue Router**

## Установка и запуск

### Требования
- Node.js 16+ 
- npm или yarn

### Установка

1. Клонируйте репозиторий:
```bash
git clone <repository-url>
cd <project-folder>
```

2. Установите зависимости:
```bash
npm install
# или
yarn install
```

3. Запустите проект в режиме разработки:
```bash
npm run dev
# или
yarn dev
```

4. Откройте браузер и перейдите по адресу `http://localhost:5173`

### Сборка для production
```bash
npm run build
# или
yarn build
```

## Особенности реализации

1. **Валидация в реальном времени**:
   - Текстовые поля валидируются при потере фокуса
   - Селекты валидируются при изменении значения
   - Обязательные поля подсвечиваются красной обводкой при ошибке

2. **Сохранение состояния**:
   - Используется Pinia с сохранением в localStorage
   - Состояние сохраняется при обновлении страницы

3. **Обработка меток**:
   - Метки преобразуются из строки `"метка1;метка2"` в массив `[{text: "метка1"}, {text: "метка2"}]`
   - Обратное преобразование при редактировании

4. **Условные поля**:
   - Поле "Пароль" динамически скрывается/показывается в зависимости от выбранного типа записи

## Использование

1. **Добавление учетной записи**:
   - Нажмите кнопку "+" в правом верхнем углу
   - Заполните необходимые поля
   - Поля автоматически сохраняются после валидации

2. **Редактирование учетной записи**:
   - Измените значение в любом поле
   - Изменения сохраняются автоматически после валидации

3. **Удаление учетной записи**:
   - Нажмите на иконку корзины рядом с учетной записью
   - Запись будет немедленно удалена

4. **Работа с метками**:
   - Введите метки через точку с запятой: `работа;админ;тест`
   - При сохранении они преобразуются в структурированный формат

## Валидация

- **Логин**: обязательное поле, максимум 100 символов
- **Пароль**: обязательное для типа "Локальная", максимум 100 символов
- **Тип записи**: обязательное поле
- **Метка**: необязательное поле, максимум 50 символов
